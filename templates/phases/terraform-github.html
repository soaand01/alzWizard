Phase 2: GitHub + Terraform

Step-by-step adapted guidance for bootstrapping GitHub for the ALZ Terraform flow. This page summarizes the official ALZ instructions and links to the authoritative docs for full details.

Folder layout
mkdir -p ~/accelerator/config
mkdir -p ~/accelerator/output
touch ~/accelerator/config/inputs.yaml
# optional for Terraform AVM starter
touch ~/accelerator/config/platform-landing-zone.tfvars
If using ALZ AVM starter modules

Create a lib folder under config and copy the starter lib templates into it (example using a temporary clone):

tempFolder=~/accelerator/temp
mkdir -p "$tempFolder"
git clone -n --depth=1 --filter=tree:0 "https://github.com/Azure/alz-terraform-accelerator" "$tempFolder"
cd "$tempFolder"
libFolderPath="templates/platform_landing_zone/lib"
git sparse-checkout set --no-cone "$libFolderPath"
git checkout
cp -r "$tempFolder/$libFolderPath" ~/accelerator/config/
cd ~
rm -rf "$tempFolder"

Run the bootstrap

Use the ALZ PowerShell module (pwsh) to run the bootstrap. Install or update the module first:

# in pwsh
Get-InstalledModule -Name ALZ || Install-Module -Name ALZ -Scope CurrentUser -Force
Update-Module -Name ALZ

Then run the bootstrap (example):

Import-Module ALZ
Start-ALZBootstrap -ConfigFile ~/accelerator/config/inputs.yaml -OutputPath ~/accelerator/output
# or using Deploy-Accelerator for Terraform starter modules
Deploy-Accelerator -inputs "~/accelerator/config/inputs.yaml" -starterAdditionalFiles "~/accelerator/config/lib" -output "~/accelerator/output"
Verify
  • Confirm the bootstrap completes and a starter repository appears in your GitHub organization.
  • Check Actions workflows in the repository; initial runs should complete or show the expected artefacts.
  • Ensure terraform.tfvars.json appears if generated for the starter module.

Source: adapted from the official ALZ docs: GitHub + Terraform

What this step will do — GitHub + Terraform

This step prepares and runs the ALZ Terraform bootstrap flow using GitHub as the CI/CD host. In short, it will:

  • Create or populate a starter repository in your GitHub organization containing the ALZ starter code for the selected platform.
  • Provision the required pipeline/workflow files and secrets (e.g., repository secrets, PATs) so the bootstrap can execute safely from GitHub Actions.
  • Execute the bootstrap logic (via Actions) which will generate environment configuration artifacts, initial Terraform state, and starter resources required by the ALZ pattern.

What to expect and check:

  • After a successful run you should see a starter repo in your organization and one or more Actions workflow runs.
  • The process may create or update infrastructure in the target subscription when the pipelines run—ensure you use the correct subscription and credentials.
  • Keep the generated terraform.tfvars.json and any outputs safe; they contain environment configuration for later stages.

Tip: Review the example inputs-github.yaml before running; it controls naming, identities, and the bootstrap behavior.
